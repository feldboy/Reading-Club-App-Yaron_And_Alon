פרויקט סיום קורס

יש לפתח צד שרת (node.js) וצד לקוח (אפליקצית React)
אפשר לבצע את הפרויקט בקבוצות של בין סטודנט אחד ל- 4, תכולת הפרויקט וההקפדה בהגנה תלויה בגודל הקבוצה.
	
תכולה הפרויקט:

התחברות: 
הרשמה ולוגאין של משתמש לפי שם משתמש וסיסמה
אפשרות הרשמה ע״י ספק חיצוני (google או facebook)
יש להשתמש ב- JWT כולל Refresh token כפי שנלמד בכיתה (בנוסף לספק חיצוני).
האפליקציה צריכה לזכור משתמש שהתחבר.
לאפשר למשתמש גם להתנתק

מסך פרטי משתמש:
מציג את פרטי המשתמש כולל תמונה
הצגת הפוסטים של אותו משתמש
עבור המשתמש המחובר תהיה אפשרות עריכה (של התמונה ושל שם המשתמש, אימייל וסיסמה אין צורך לעדכן).

AI:
האפליקציה תשתמש באחד ממקורות ה- AI הזמינים: גמיני, chatGPT, המודל של המכללה… המידע צריך להיות רלונטי לאפליקציה.
יש לשים לב להתממשקות מול ה- API כך שלא תהיה חריגה במספר הבקשות!
*** יש להשתמש ב- AI כדי לנתח או להציג תכנים בצורה חכמה מהאפליציה, כלומר לא מספיק לבצע קריאה ל-AI וזהו. למשל יש לממש חיפוש מידע בלשון חופשית באפליקציה כמו שהודגם בכיתה. יש לשים לב לארכיטקטורת ה- backend.

שיתוף תכנים:
משתמש יוכל להעלות מידע לאפליקציה הכולל:
טקסט
תמונה
משתמשים אחרים יוכלו לראות את המידע שהועלה
המשתמש שהעלה יוכל לעדכן את המידע שהוא פרסם גם את התמונה וגם את הטקסט	
המשתמש שהעלה יוכל למחוק את המידע שהוא פרסם
כל משתמש יוכל לראות את התכנים שהוא העלה (מסך נפרד או סינון)
הצגת התכנים בגלילה עם paging (טעינה מדורגת, לא דפים נפרדים)
תמונת פרופיל ושם משתמש לא נחשב לשיתוף תמונה/טקסט

יש להוסיף אפשרות למשתמשים להגיב על התכנים של משתמשים אחרים, התגובות יוצגו במסך נפרד ולא במסך הראשי, כאשר במסך הראשי יופיע מספר התגובות שנכתבו על אותו תוכן.

יש לאפשר למשתמש לסמן פוסטים נבחרים (like) וכמובן להציג את הסימון… יש להתיחס למימוש מתאים ויעיל ב- DB.

חובה לעבוד עם גיט, כל משתמש עם הפרופיל שלו, לעשות קומיטים קטנים. לעבוד עם pull request כפי שנדרשתם לבצע בתרגילים. חלוקת הקוד בין חברי הקבוצה צריכה להיות סבירה!

יש לתעד את כל ה- API באמצעות swager

יש לכתוב unit-test עבור כל ה- api של השרת (פרט לשימוש ב- API החיצוני).

יש להשתמש ב- typescript בלבד

תמונות יש לשמור בשרת שלכם, לא ב- DB ולא בשרות חיצוני

 יש להשתמש במונגו מקומי (בשרת של המכללה) בלבד (לא אטלס או שירותי ענן אחרים).

יש להעלות את האפליקציה לשרת של המכללה שיהיה נגיש מכל מקום דרך URL.
הפעלת האפליקציה עם https (גם ה- backend וגם ה- frontend)
האפליקציה צריכה לעבוד ברקע גם אם הטרמינל שלכם סגור - יש להשתמש ב- PM2
האפליקציה חייבת לרוץ במוד production 
node_env = production
יש להשתמש במונגו על השרת ולא בשרותי ענן כאשר ההתחברות עם מונגו מוגנת ע״י שם משתמש וסיסמה.
עליכם לדאוג שהכניסה לאפליקציה תתבצע דרך הדומין שקיבלתם מדורון - כלומר אני מקליד את שם הדומיין (ללא פורט) והאפליקציה נפתחת אצלי במחשב ללא VPN.
אסור בכל מקרה להשתמש בשרותי hosting חיצוניים

עיצוב, יש לדאוג לעיצוב המסכים, שימוש בצבעים, רקעים, ניצול שטח המסך… עיצוב לא טוב יגרור הורדת ציון.
